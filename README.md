# Лабораторная работа №1

Система мониторинга транзакций в реальном времени. Проект демонстрирует использование Apache Kafka для создания отказоустойчивого конвейера данных, где каждая транзакция проверяется моделью машинного обучения

##Состав репозитория

Producer: Имитирует поток транзакций, считывая данные из CSV и отправляя их в Kafka в формате JSON.

Kafka Cluster: Хранит и распределяет сообщения между потребителями. Состоит из Zookeeper и 2-х брокеров.

Consumer: Загружает обученную модель и энкодеры (.pkl), выполняет препроцессинг и выводит результаты предсказаний в логи.

Dashboard: Веб-интерфейс на Streamlit для визуального контроля системы.

### Подготовка данных

Так как основной датасет transactions.csv имеет большой объем и не хранится в репозитории, его необходимо сгенерировать перед первым запуском системы при помощи команды: 

python data/generator.py

### Для запуска программы требуется выполнить следующую команду в терминале: docker-compose up --build

### Дэшборд находится по пути: http://localhost:8501
